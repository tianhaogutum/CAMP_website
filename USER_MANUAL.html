Overview

The CAMP (Center for Entrepreneurial and Financial Studies) website is a modern, responsive multi-page application built with React, Tailwind CSS, and Relume UI components. This manual provides comprehensive guidance for both developers and end-users on how to effectively use and maintain the website.

🌐 Website Features

  • • Multi-page application (Home, Projects, Team, Contact)
  • • Responsive design for all devices
  • • Dynamic content management via CSV files
  • • Modern UI with smooth animations
  • • Professional research presentation

🛠 Technical Stack

  • • React 18 with modern hooks
  • • Tailwind CSS for styling
  • • Relume UI components
  • • React Router for navigation
  • • Framer Motion for animations

For Developers

Project Architecture

This project is built on React with HTML, CSS, and utilizes Relume-supported libraries for modern UI components. The architecture follows a component-based approach with clear separation of concerns. 

📁 Project Structure:
├── src/
│   ├── components/          # React components
│   │   ├── home/           # Homepage components
│   │   ├── project/        # Individual project pages
│   │   ├── projects/       # Projects listing page
│   │   ├── team/           # Team page components
│   │   └── contact/        # Contact page components
│   ├── utils/              # Utility functions
│   │   ├── projectData.js  # Project data management
│   │   └── teamData.js     # Team data management
│   └── App.jsx             # Main application
├── public/
│   ├── data/               # CSV data files
│   │   ├── projects.csv    # Project information
│   │   └── team.csv        # Team member information
│   └── images/             # Image assets
└── package.json            # Dependencies and scripts

For Users

Adding New Projects

📋 Quick Guide: To add a new project, edit the public/data/projects.csv file following the existing format and examples. The system automatically reads CSV files and updates the website content dynamically.

Step-by-Step Process

  1. Open the CSV file: Navigate to public/data/projects.csv
  2. Study existing examples: Review the current project entries to understand the format
  3. Add new row: Create a new row with all required fields
  4. Fill in details: Complete all fields with accurate information
  5. Check spelling: Ensure no typos or formatting errors
  6. Save the file: The website will automatically update
  7. Refresh browser: View the changes on the website

Complete Field List

Project ID,Title,Description,Short Description,Researchers,
Asset Class,Data Used,Advanced Method,Tags,
Content Paragraph 1,Content Paragraph 2,Content Paragraph 3,
Introduction,Methodology,Key Findings,Policy Implications,
Conclusion,References,Appendices,Author Note

Field Descriptions & Special Requirements

Content Fields (Support Rich Text)
  • • Introduction: Project introduction section, supports multiple paragraphs
  • • Methodology: Research methodology section, supports lists and paragraphs
  • • Key Findings: Main research findings, supports multiple paragraphs
  • • Policy Implications: Policy impact analysis, supports paragraph format
  • • Conclusion: Research conclusions, supports paragraph format
  • • References: Academic references, supports multiple lines
  • • Appendices: Supporting materials, supports list format
  • • Author Note: Author information, supports links and additional notes
⚠️ Critical Requirements:
  • • File Encoding: Ensure CSV file uses UTF-8 encoding
  • • Unique IDs: Use unique Project IDs (increment from existing numbers)
  • • Comma Handling: Fields containing commas must be wrapped in double quotes
  • • Special Characters: Fields with quotes or line breaks need double quote wrapping
  • • Researcher Names: Must match exactly with team member names in team.csv
  • • Field Completeness: All required fields should be filled

Adding Team Members

Process Overview

  1. Edit team CSV: Open public/data/team.csv
  2. Add member information: Fill in all required fields
  3. Add photo: Place image file in public/images/ directory
  4. Update CSV reference: Reference the image file in the CSV
  5. Save and refresh: Changes appear automatically

Complete Field List

Person ID,Name,Title,Role,Description,Image,Category,
Email,LinkedIn,Website,Research Areas,Education,Experience

Special Field Requirements

Critical Fields
  • • Category: Must be exactly "Research Leads" or "Research Assistants"
  • • Image: Photo filename, stored in public/images/ directory
  • • Person ID: Unique identifier (increment from existing numbers)

Image Requirements

  • • Format: JPG, PNG, or WebP
  • • Size: Recommended 400x400px or square aspect ratio
  • • Location: Save in public/images/ directory
  • • Naming: Use descriptive names (e.g., john-doe.jpg)
  • • Reference: Use /images/filename.ext in CSV
✅ Category Options:
  • • "Research Leads" - For principal researchers and professors
  • • "Research Assistants" - For research assistants and project managers
⚠️ Data Consistency Requirements:
  • • File Encoding: Ensure CSV file uses UTF-8 encoding
  • • Unique IDs: Person IDs must be unique across all team members
  • • Image Files: Referenced image files must exist in the images directory
  • • Name Matching: Names used in projects.csv must match exactly with team.csv

⚠️ Critical Requirements Checklist

For Projects:

  • □ Unique Project ID (increment from existing)
  • □ All required fields filled
  • □ Researcher names match team.csv exactly
  • □ UTF-8 encoding used
  • □ Commas in text fields wrapped in quotes

For Team Members:

  • □ Unique Person ID
  • □ Category is "Research Leads" or "Research Assistants"
  • □ Image file exists in public/images/
  • □ Image path correctly referenced
  • □ All contact information provided
